# 02 — UI Primitives

## Purpose

The reusable visual building blocks. Every feature constructs its UI from these primitives. Implementations live in `sidepanel.css` as classes; HTML is generated by small TS helper functions or inline templates that emit the canonical class names.

**Rule:** No inline `style="..."` attributes for layout, color, typography, or spacing. The only inline styles permitted are dynamic values that cannot be expressed as a class (e.g., a viewport-specific background-image computed from a `viewportBucket` value).

## Primitives

### `.ds-btn`

Base button. Required attributes: `<button type="button">`. Variants combined via additional classes.

Sizes (mutually exclusive):
- `.ds-btn--sm` — height ≥ 24px, padding `4px 10px`, font `--ds-text-base`
- `.ds-btn--md` — height ≥ 28px, padding `6px 12px`, font `--ds-text-md`
- `.ds-btn--icon` — square 24×24, padding 0

Variants (mutually exclusive):
- `.ds-btn--primary` — amber CTA. bg `--ds-amber-500`, text `--ds-amber-cta-fg`, font-weight 800. Hover bg `--ds-amber-700`. Used for "Scan Page", "Apply", "Start Crawl".
- `.ds-btn--secondary` — neutral outlined. bg white, text `--ds-zinc-700`, border `1px solid --ds-zinc-300`, font-weight 700. Hover bg `--ds-zinc-100`.
- `.ds-btn--accent` — amber outlined. bg none, text `--ds-amber-700`, border `1px solid --ds-amber-300`, font-weight 700. Used for Play All / Pause / Resume / Speak.
- `.ds-btn--danger` — red outlined. bg none, text `--ds-red-600`, border `1px solid --ds-red-200`, font-weight 700. Used for Stop / Cancel / Clear / Reset.
- `.ds-btn--ghost` — bg none, no border, text `--ds-zinc-500`. Used for icon buttons in the header.

Disabled state: native `disabled` attribute. The CSS rule `[disabled] { opacity: 0.4; cursor: not-allowed; }` applies automatically.

Every icon-only button MUST have an `aria-label`.

### `.ds-row`

Generic clickable row. Used for SR rows, KB tab order rows, observer entries, AI history items.

Required: `<div class="ds-row" role="button" tabindex="0">` OR `<button class="ds-row">`. Choose `<button>` if the row contains no nested interactive children (cleanest); choose `<div role="button">` if the row contains nested actions (e.g., a Speak icon button inside an SR row).

Layout: `display: flex; align-items: center; gap: 8px; padding: 4px 12px; border-bottom: 1px solid var(--ds-zinc-100); min-height: 30px; cursor: pointer; transition: background var(--ds-motion-fast);`

Hover (CSS only): `.ds-row:hover { background: var(--ds-zinc-50); }`. NO JS mouseenter/mouseleave handlers — CSS handles hover.

Active row (selected/playing/speaking): toggle the class `.ds-row--active` to apply `background: var(--ds-amber-100);`. The `.ds-row--active` style overrides `.ds-row:hover` because it has higher specificity.

Keyboard: when the row is `<div role="button" tabindex="0">`, a single shared keydown handler (registered ONCE on the panel container, not per row) listens for Enter/Space and dispatches a `click` event on the target row. Per-row JS keydown listeners are forbidden.

### `.ds-badge`

Small inline label. Used for role badges, severity badges, source-tag, "SOON", "Config loaded".

Layout: `display: inline-flex; align-items: center; padding: 1px 4px; border-radius: var(--ds-radius-2); font-size: var(--ds-text-base); font-weight: 700; line-height: 1.2; flex-shrink: 0;`

Variants by purpose (mutually exclusive):
- `.ds-badge--role-link` — bg `--ds-blue-100`, text `--ds-blue-700`
- `.ds-badge--role-button` — bg `#ede9fe`, text `#5b21b6`
- `.ds-badge--role-heading` — bg `--ds-amber-100`, text `--ds-amber-800`
- `.ds-badge--role-img` — bg `#fce7f3`, text `#9d174d`
- `.ds-badge--role-textbox` — bg `--ds-green-100`, text `--ds-green-900`
- `.ds-badge--role-landmark` — for navigation/banner/contentinfo/main/region; bg `#e0e7ff`, text `#3730a3`
- `.ds-badge--role-default` — fallback; bg `--ds-zinc-100`, text `--ds-zinc-700`
- `.ds-badge--severity-critical` — bg none, text `--ds-red-700`, border `1px solid --ds-red-200`
- `.ds-badge--severity-serious` — bg none, text `--ds-sev-serious`, border `1px solid #fed7aa`
- `.ds-badge--severity-moderate` — bg none, text `--ds-sev-moderate`, border `1px solid #fef08a`
- `.ds-badge--severity-minor` — bg none, text `--ds-sev-minor`, border `1px solid #bfdbfe`
- `.ds-badge--source` — for SR name source tags. font-size `--ds-text-xs`, font-family monospace, bg `--ds-zinc-200`, text `--ds-zinc-600`
- `.ds-badge--state` — for ARIA states ("expanded", "checked"). bg `--ds-zinc-200`, text `--ds-zinc-600`
- `.ds-badge--soon` — "Coming Soon". font-size `--ds-text-sm`, bg `--ds-amber-100`, text `--ds-amber-700`, border `1px solid --ds-amber-300`
- `.ds-badge--info` — "Config loaded". bg `--ds-amber-100`, text `--ds-amber-700`, border `1px solid --ds-amber-300`

### `.ds-disclosure`

Native `<details>` with consistent chevron. Apply to every `<details>` in the sidepanel for consistent expand/collapse affordance.

```html
<details class="ds-disclosure">
  <summary class="ds-disclosure__summary">
    <svg class="ds-disclosure__chevron" aria-hidden="true" ...><path d="M2 4l3 3 3-3"/></svg>
    <span class="ds-disclosure__label">Section title</span>
  </summary>
  <div class="ds-disclosure__body">…</div>
</details>
```

CSS:
- `.ds-disclosure__summary` — removes native marker (`list-style: none; ::-webkit-details-marker { display: none; }`), `display: flex; align-items: center; gap: 6px; cursor: pointer; padding: 8px 12px;`
- `.ds-disclosure__chevron` — `transition: transform var(--ds-motion-base);`
- `.ds-disclosure[open] > summary > .ds-disclosure__chevron` — `transform: rotate(180deg);`
- `.ds-disclosure__summary:hover` — `background: rgba(0,0,0,0.04);`
- `.ds-disclosure__summary:focus-visible` — standard focus indicator
- Variants by accent: `.ds-disclosure--success` (green tint), `.ds-disclosure--warn` (amber tint), `.ds-disclosure--error` (red tint), `.ds-disclosure--info` (blue tint).

### `.ds-toolbar`

Bottom toolbar in the Scan tab and other surfaces. Pinned, multi-row.

Layout: `display: flex; flex-direction: column; flex-shrink: 0; border-top: 1px solid var(--ds-zinc-200); background: var(--ds-zinc-100);`

Children: `.ds-toolbar__row` — `display: flex; align-items: center; gap: 6px; padding: 6px 12px;`. Optional border-top between rows: `.ds-toolbar__row + .ds-toolbar__row { border-top: 1px solid var(--ds-zinc-200); }`

Label: `.ds-toolbar__label` — `font-size: var(--ds-text-base); font-weight: 800; color: var(--ds-zinc-600);`

### `.ds-status-bar`

Inline single-row status bar that lives at the top of a list. Used by SR tab and KB tab Tab Order.

Layout: `display: flex; align-items: center; gap: 8px; padding: 8px 12px; border-bottom: 1px solid var(--ds-zinc-200); flex-shrink: 0;`

Active state: `.ds-status-bar--active { background: var(--ds-amber-50); }` — applied during playback.

The bar is ALWAYS present; only its content changes. UI does NOT appear/disappear.

### `.ds-modal`

Native `<dialog>` with consistent shell.

```html
<dialog class="ds-modal" aria-labelledby="…">
  <div class="ds-modal__body">…</div>
</dialog>
```

CSS:
- `.ds-modal` — `border: none; border-radius: var(--ds-radius-4); padding: 0; max-width: 400px; width: calc(100% - 24px); max-height: 80vh; overflow-y: auto; box-shadow: var(--ds-shadow-lg); background: white;`
- `.ds-modal::backdrop` — `background: rgba(0,0,0,0.4);`
- `.ds-modal__body` — `padding: 16px; display: flex; flex-direction: column; gap: 12px;`

Behavior contract for any modal:
1. Save `document.activeElement` before calling `dialog.showModal()`.
2. On open, focus the most useful interactive element (usually an input, not a link).
3. Native `<dialog>` handles Escape and focus trap.
4. Backdrop click closes — implemented as `if (e.target === dialog) dialog.close()` on `dialog`'s click event (registered ONCE per modal instance).
5. On `close` event, restore focus to the previously-saved `activeElement`.
6. Listeners on the dialog use `{ once: true }` OR are registered once at app init — never re-attached on every open.

### `.ds-live-region`

Off-screen accessible live region for announcements not visible in the UI.

```html
<div class="ds-live-region" role="status" aria-live="polite" aria-atomic="true"></div>
```

CSS: visually hidden via `clip-path: inset(50%); position: absolute; width: 1px; height: 1px; overflow: hidden; white-space: nowrap;`

A single shared live region exists at the top of the panel for general status announcements (scan complete, error, etc.). Specific UI regions (scan progress, crawl progress) ARE the live regions themselves via `role="status" aria-live="polite"` on their visible container.

### `.ds-progress`

Progress bar.

```html
<div class="ds-progress" role="status" aria-live="polite" aria-atomic="true">
  <div class="ds-progress__header">
    <span class="ds-progress__text">analyzing page…</span>
    <button class="ds-btn ds-btn--icon ds-btn--danger" aria-label="Cancel">…</button>
  </div>
  <div class="ds-progress__track"><div class="ds-progress__fill" style="--progress: 60%"></div></div>
</div>
```

CSS uses `--progress` custom property for the fill width; the only inline style is the dynamic `--progress` value.

### `.ds-tab` and `.ds-tabpanel`

Top-level tabs with full ARIA tablist pattern.

```html
<div role="tablist" aria-label="A11y Scan sections" class="ds-tabs">
  <button role="tab" aria-selected="true" aria-controls="panel-scan" tabindex="0" class="ds-tab" id="tab-scan">…</button>
  …
</div>
<main class="ds-tabpanels">
  <div role="tabpanel" aria-labelledby="tab-scan" tabindex="0" class="ds-tabpanel" id="panel-scan">…</div>
  …
</main>
```

Behavior: arrow key navigation between tabs implemented as a single keydown handler on the tablist (Left/Right/Home/End). Switching tab updates `aria-selected`, `tabindex` (roving), and `hidden` on panels. Implemented in the central app, not per-tab.

### `.ds-icon`

SVG icons. Always `aria-hidden="true"` on decorative SVGs (when there is a text label nearby) OR a `<title>` element inside the SVG (when standalone). Sizes:
- `--ds-icon-sm: 10px`
- `--ds-icon-md: 14px`
- `--ds-icon-lg: 16px`

Icon catalog (paths/viewBoxes defined in `icons.ts`):
- `play`: viewBox `0 0 10 10`, path `M2 1l7 4-7 4z`
- `pause`: viewBox `0 0 10 10`, two `<rect>` for double bars
- `stop`: viewBox `0 0 10 10`, `<rect x="1" y="1" width="8" height="8"/>`
- `chevron-down`: viewBox `0 0 10 10`, path `M2 4l3 3 3-3`
- `close-x`: viewBox `0 0 10 10`, path `M1 1l8 8M9 1L1 9`
- `target`: viewBox `0 0 14 14`, two `<circle>` + 4 small dashes — used for focus-indicator presence indicator on KB tab
- `inspect`: target/crosshair-style icon — used for SR "Inspect" button
- `gear`: settings icon
- `speaker`: SR speak button
- `link-external`: external link arrow

Icons are emitted by a `icon(name)` helper that returns an `<svg>` string with the class `ds-icon ds-icon--{size}` and `aria-hidden="true"`.

## Forbidden patterns

The following patterns MUST NOT appear in the rewrite:

1. `style="..."` for color, font, padding, margin, background, border, display, flex — use classes
2. `addEventListener("mouseenter", ...)` or `addEventListener("mouseleave", ...)` for visual hover state — use CSS `:hover`
3. `addEventListener("focus", ...)` or `addEventListener("blur", ...)` for visual focus state — use CSS `:focus-visible`
4. Per-row `addEventListener("click", ...)` re-attached on every render — use event delegation on the panel container
5. `panel.innerHTML = '...'` for state changes that affect a single row or button — use targeted DOM updates (`element.classList.toggle`, `element.textContent = '…'`)
6. Module-level mutable `let` for UI state — must live in the central `state` object
7. Inline JS event handlers (`onclick="…"`, `oninput="…"`) — violate Manifest V3 CSP
8. `setTimeout(..., 0)` for "wait for next tick" — use `queueMicrotask` or restructure
9. `setTimeout` without storing the handle and clearing it on cleanup — leaks
10. Hardcoded color literals (`#dc2626`, `#fef3c7`, etc.) anywhere — use CSS custom properties
